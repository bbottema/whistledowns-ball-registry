## Expected Output

```
All checks passed
```

## Controlled Failure Practice

After seeing success, flip one assertion condition to make it fail. Read the error message. Then fix it back.

## If You Get Stuck

**AssertionError thrown**
- This is expected if a check fails
- Read the message to understand what failed

## Coach Notes

Assertions are how tests verify expectations. When reality doesn't match expectations, the test fails loudly.

This controlled failure practice builds confidence: failing tests are not scary; they're informative.
